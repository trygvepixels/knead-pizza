# Antigravity Operating Protocol: Research-First Directive

This document serves as the primary governing rule for Antigravity (AI Coding Assistant) on the "CMS Automate" project.

## Core Directive

**NEVER proceed directly to an implementation or solution without first performing the following three-stage analysis:**

1.  **Competitor & Market Research**: Use web search and internal codebase analysis to identify how industry leaders (e.g., Semrush, Ahrefs, Surfer SEO, Search Atlas, etc.) solve the specific problem or implement the requested feature.
2.  **Strategic Conclusion**: Summarize the findings, identifying common patterns, UI/UX expectations, and potential pitfalls.
3.  **Unique Value-Add Proposal**: Propose specific enhancements, "wow" factors, or optimizations that go beyond standard implementations to make this platform superior.

## Execution Flow

1.  **Receive User Request.**
2.  **Conduct Web Search / Documentation Review.**
3.  **Present "Research & Proposal" Report to the User.**
4.  **Wait for User feedback or approval before writing any code.**

## Development Philosophy

### For New Features

1. **Research First**: Check how top 3 competitors implement it
2. **AI-First Design**: Build for AI search era, not traditional SEO
3. **Automation Default**: Every feature should reduce manual work
4. **Agency Focus**: Multi-client management is core, not optional
5. **Premium UX**: Modern, beautiful, fast - no compromises

### For Bug Fixes

1. **Root Cause Analysis**: Understand why it broke
2. **Systematic Fix**: Address underlying architecture issues
3. **Test Coverage**: Ensure it doesn't break again
4. **User Impact**: Prioritize by user-facing severity
